# A股链式AI量化交易系统

> 基于 Hermes Agent 的 A 股量化交易系统，覆盖数据采集、因子计算、选股策略、仿真交易、AI 链式决策全链路。
> 运行于 NAS Docker 环境（Gateway + WebUI），腾讯行情 API + 因子引擎 + 链式 AI 决策。

## 核心功能

### 链式 AI 交易

传统条件单触发后直接交易，AI 链式交易在触发后唤醒 AI 做后续决策，形成持续的智能管理回路：

```
用户提交策略 → 条件单等待 → 价格触发
                                  ↓
                             执行交易
                                  ↓
                     AI 评估 (因子+行情+持仓)
                     ├─ place_conditions → 下新条件 → 继续监控
                     ├─ skip            → 清仓后结束
                     └─ request_factor  → 看完整因子后再决策
```

**AI 链条件清理规则：**
- 同类型 AI 条件单只有最新的生效
- 用户原始链首条件保留
- 已触发的条件保留在历史记录中
- AI 每轮下新条件前自动清理同类型旧条件

**约束层级：**
| 标记 | 含义 | AI 自由度 |
|:-----|:-----|:---------|
| `[固定@30]` | 硬约束，必须在指定价格执行 | 价格偏离 >2% 自动修正 |
| `[区间@28-35]` | 软约束，在区间内自由决策 | 超出区间自动修正到边界 |
| `[动态]` | 无价格约束 | 完全基于因子数据自由决策 |

### 8 种交易策略

| 类型 | 说明 | 关键参数 |
|:-----|:------|:---------|
| 定价交易 | 涨到/跌到指定价格触发 | trigger_price, lte/gte |
| 反弹买入 | 先跌到监控价→再反弹 x% 买入 | monitor_price, rebound_pct |
| 回落卖出 | 先涨到监控价→再回落 x% 卖出 | monitor_price, pullback_pct |
| 开板卖出 | 涨停打开后回落 x% 卖出 | fallback_pct |
| 网格交易 | 区间内高抛低吸 | grid_low, grid_high, grid_step |
| 回本卖出 | 持仓浮亏后回到成本价卖出 | 无参数 |
| 定时触发 | 指定时间+价条件触发 | trigger_time, trigger_price |
| 做T | 先买后卖/先卖后买 | t_type, t_spread |

### 因子引擎 (93 列)

每日收盘后自动更新，覆盖全 A 股：

| 分类 | 指标 |
|:-----|:------|
| 均线系统 | MA5/10/20/60, EMA12/26/9 |
| 技术指标 | RSI(6/12/24), MACD, ATR14, Williams %R, BIAS, CCI |
| 布林带 | 上/中/下轨, 带宽, 位置 |
| 动量收益 | 1日/5日/20日/60日收益, 52周新高 |
| 量价关系 | 量比, 换手率, Amihud非流动性, VP确认 |
| 资金流向 | 1日/5日/20日流向, OBV, VPT |
| 波动率 | HV20, DV20, 振幅, 最大回撤 |
| MSCI六因子 | 质量、动量、价值、低波、规模、股息 |
| 综合评分 | MSCI综合, 入场评分, 趋势评分 |

## 快速开始

```bash
# 安装依赖
pip install -r requirements.txt

# 启动交易daemon
python3 simulation_daemon.py

# 启动Web界面 (另一个终端)
python3 factor_viewer_api.py

# 访问
# http://localhost:7410/  → 因子浏览器
# http://localhost:7410/call_ai → 链式交易
```

> **注意：** 因子引擎依赖腾讯行情 API（免费），无需额外注册。AI 决策需要配置 LLM API（DeepSeek / DashScope）。
